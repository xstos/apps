If you add 1000 text blocks to a flow layout panel in WPF (or any other GUI framework for that matter), the UI seizes up. This necessitates brittle and complex virtualization models and other shenanigans. This is a from-scratch rendering system proof of concept to address those issues. It's effectively a dynamic infinitely-nestable document tree. First version will use software rendering to reduce complexity. The next version will potential use the GPU for speedups (i.e. like the batching in this prototype https://github.com/xstos/apps/tree/master/playground/js/KriterisEngine/src/experimental/gpuedit )
